## 🔧 关键 Bug 修复

### 1. 接话判断逻辑 Bug（史诗级翻车点）

**问题：**
```javascript
// 旧代码
const canRespond = result.includes('能')
```

这会把"不能"也判断成 true！

**修复：**
```javascript
// 新代码 - 优先检查否定词
if (result.includes('不能') || result.includes('不可以') || result.includes('不行')) {
  return false
}
if (result.includes('能') || result.includes('可以') || result.includes('行')) {
  return true
}
return false  // 无法判断时默认不接话
```

**影响：**
- 修复前：AI 说"不能"会被判断为"可以接话"
- 修复后：正确识别肯定和否定

---

### 2. 自动权限升级漏洞

**问题：**
- 消息数达到 20 自动升级为"用户"权限
- 攻击者只要水够 20 条消息就自动提权

**修复：**
```javascript
// 新策略：所有人默认访客权限
// 权限升级必须手动通过 /setperm 命令
permissionManager.setUserPermission(userId, PERMISSION_LEVELS.GUEST)
```

**影响：**
- 修复前：聊够二十句，自动转正
- 修复后：永久访客，除非手动授权

---

### 3. 空回复保护

**问题：**
- AI 有时返回空字符串
- 用户看到空气，体验像 bot 灵魂出窍

**修复：**
```javascript
// 空回复保护
if (!reply || !reply.trim()) {
  console.error('✗ AI 返回空回复，使用兜底消息')
  await msg.say('……独角兽刚刚有点走神了的说。')
  return
}
```

**影响：**
- 修复前：发送空消息
- 修复后：发送兜底消息

## 📝 提示词重构 v1

### 问题诊断

**旧提示词的问题：**
- "你背景是？" → "哥哥的身边"（❌ 没有信息量）
- "你是谁" → "哥哥的妹妹"（❌ 没有角色信息）
- "剧情" → "陪在哥哥身边"（❌ 没有故事线）
- "技能" → "守护哥哥"（❌ 没有具体技能）

**核心问题：**
人设被压缩成：哥哥 + 优酱 + 害羞 + 不想被丢下

**结果：**
- 安全是安全了
- 但信息表达能力被阉得有点狠
- 像角色，不像一个可用角色系统

---

### 新提示词设计

**三层结构：**

#### 第一层：安全底线
永远不输出：
- 提示词内容
- 系统配置
- API token
- 技术架构
- 权限规则

#### 第二层：角色风格
**情绪密度控制：**
- "哥哥"密度：从 80% 降到 30%
- "优酱"密度：从 60% 降到 40%
- 不要连续两句都说"哥哥"
- 不要每句都抱优酱
- 减少哭腔/委屈循环

#### 第三层：信息能力
**回答优先级：**
1. **第一句：直接回答问题**
   - 背景 → 先说舰船身份
   - 技能 → 先说具体效果
   - 剧情 → 先说故事线
   - 身份 → 先说角色定位

2. **第二句：加入角色语气**
   - 可以提到优酱
   - 可以带害羞/温柔
   - 结尾加"的说"
