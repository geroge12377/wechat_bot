# 智能对话启动系统

## 📋 功能概述

实现了智能的对话启动机制，让机器人能够：
1. 自动检测能否接话
2. 智能判断是否回应
3. 待机状态管理
4. 30秒超时提醒
5. 自动权限分配

---

## 🔄 工作流程

### 1. 消息接收
```
用户发送消息
    ↓
白名单检查
    ↓
自动分配权限（基于好感度和消息数）
    ↓
判断是否为命令
```

### 2. 关键词检查
```
包含关键词？
├─ 是 → 清除状态 + 退出待机 → 正常对话
└─ 否 → 检查状态
```

### 3. 无关键词消息处理

#### 第1次无关键词消息
```
检查能否接话（使用 Gemini AI）
    ↓
能接话？
├─ 能 → 正常对话
└─ 不能 → 静默等待
```

#### 第2-3次无关键词消息
```
进入待机状态
    ↓
启动30秒超时定时器
    ↓
不回复消息
```

#### 待机状态
```
忽略所有无关键词消息
    ↓
等待关键词激活
    或
30秒后发送提醒
```

## 📊 状态管理

### 用户状态结构
```javascript
{
  noKeywordCount: 0,      // 无关键词消息计数
  isStandby: false,       // 是否处于待机状态
  lastMessageTime: null,  // 最后消息时间
  timeoutTimer: null      // 超时定时器
}
```

### 状态转换
```
正常状态
    ↓ (无关键词消息 × 1，不能接话)
等待状态
    ↓ (无关键词消息 × 2-3)
待机状态
    ↓ (关键词消息)
正常状态
```

## 🤖 AI 接话判断

### 判断逻辑
使用 Gemini 快速分析消息，判断"独角兽"角色能否自然地接话继续对话。

### 示例

**可以接话的消息：**
- "今天天气真好" → 可以聊天气
- "我好累啊" → 可以关心安慰
- "刚看了部电影" → 可以聊电影

**不能接话的消息：**
- "我去吃饭了" → 单方面通知，无法接话
- "嗯" → 太简短，无法展开
- "好的" → 确认性回复，无需接话

## 🔐 自动权限分配

### 权限等级
- **访客 (0)**: 新用户或低活跃用户
- **用户 (1)**: 消息数 ≥ 20
- **VIP (2)**: 好感度 ≥ 80 且消息数 ≥ 100
- **管理员 (3)**: 手动设置
- **所有者 (4)**: 手动设置

### 分配逻辑
```javascript
if (新用户) {
  权限 = 访客
} else if (好感度 >= 80 && 消息数 >= 100) {
  权限 = VIP
} else if (消息数 >= 20) {
  权限 = 用户
} else {
  权限 = 访客
}
```

## 💡 设计理念

### 1. 自然交互
- 不强制要求关键词
- 能接话时自然回应
- 不能接话时保持安静

### 2. 避免打扰
- 待机状态不回复无关消息
- 只在超时时提醒一次
- 等待用户主动激活

### 3. 智能判断
- AI 判断能否接话
- 避免尴尬的强行回复
- 保持角色一致性
