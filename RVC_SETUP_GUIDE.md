## 步骤 4: 在 RVC 中训练模型

### 4.1 数据预处理
1. 在 Web 界面点击 **"训练"** 标签
2. **实验名称**：输入 `unicorn`
3. **目标采样率**：选择 `40k` 或 `48k`
4. **数据集路径**：
   ```
   C:\Users\HP\Desktop\homewwork\CHATBOT_WECHAT\so-vits-svc-4.1-Stable\dataset\44k\unicorn
   ```
5. 点击 **"处理数据"** 按钮

### 4.2 提取特征
1. 等待数据处理完成
2. 点击 **"提取特征"** 按钮
3. 选择特征提取方法（推荐 `rmvpe` 或 `harvest`）

### 4.3 开始训练
1. **训练轮数**：建议 200-500 epoch
2. **批次大小**：根据显卡显存调整（8GB 显存建议 8-12）
3. **保存频率**：每 50 epoch 保存一次
4. 点击 **"开始训练"** 按钮

### 4.4 训练参数建议
- **GPU**: 如果有 NVIDIA 显卡，确保选择 GPU 训练
- **预训练模型**: 使用默认的预训练模型可以加快收敛
- **训练时间**: 根据数据量，预计 2-6 小时

## 步骤 5: 使用训练好的模型

### 5.1 模型推理
1. 训练完成后，切换到 **"模型推理"** 标签
2. 选择你训练的模型：`unicorn`
3. 上传要转换的音频文件
4. 调整参数：
   - **音高调整**: 0（保持原音高）或根据需要调整
   - **索引比例**: 0.5-0.8（控制音色相似度）
5. 点击 **"转换"** 开始推理

### 5.2 参数说明
- **音高调整 (Pitch)**:
  - 正值：提高音高
  - 负值：降低音高
  - 男转女：+12
  - 女转男：-12

- **索引比例 (Index Rate)**:
  - 0.0：不使用索引，音色可能不稳定
  - 0.5-0.8：推荐范围，平衡音色和质量
  - 1.0：完全使用索引，音色最接近训练数据

## 常见问题

### Q1: 训练时显存不足怎么办？
- 减小批次大小（batch size）
- 降低目标采样率（从 48k 降到 40k）

### Q2: 转换后的音频有杂音？
- 增加索引比例
- 检查输入音频质量
- 尝试不同的 F0 提取方法

### Q3: 音色不像训练数据？
- 增加训练轮数
- 提高索引比例
- 确保训练数据质量和多样性
